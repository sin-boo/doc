This site documents **[Multi-user Playit](https://github.com/sin-boo/multi-user-playit)** — a [playit.gg](https://playit.gg)-optimized fork of [Multi-User for Blender](https://gitlab.com/slumber/multi-user) by Swann Martinez. Use this fork for internet hosting; use the [original docs](https://slumber.gitlab.io/multi-user/index.html) for general addon usage on a LAN.

The fork keeps the same collaborative workflow but replaces port forwarding with a playit.gg TCP tunnel. Install the addon, run the agent on the host PC, create the tunnel, host in Blender, and share the public address so others can join.

## Install the addon

No official release yet — install from the GitHub repo:

1. Clone or download [sin-boo/multi-user-playit](https://github.com/sin-boo/multi-user-playit).
2. Zip the `multi_user` folder, or use a pre-built `multi_user-*.zip` from [Releases](https://github.com/sin-boo/multi-user-playit/releases) if one is available.
3. In Blender: **Edit → Preferences → Add-ons → Install from disk** → select the zip.

Dependencies are installed into Blender's Python automatically during setup.

## Quick start

1. [Download](https://playit.gg/download) and sign in to the playit.gg agent.
2. [Subscribe](https://playit.gg/account/billing/shop/premium) if you do not already have it.
3. [Create the TCP tunnel](/playit-tunnel) with port count **3** and local port **5555** (or your Blender host port).
4. In Blender: Multi-User panel → set **Port** → **Host**.
5. Share the playit address (e.g. `my-tunnel.gl.at.ply.gg:41234`) with joiners.
